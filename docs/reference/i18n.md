# Internationalisierung (i18n)

## Grundlagen

| Setting             | Wert                                    |
| ------------------- | --------------------------------------- |
| Text-Domain         | `'resa'`                                |
| Entwicklungssprache | Deutsch                                 |
| Zielregion          | DACH (Deutschland, Österreich, Schweiz) |

## PHP Backend

```php
// Standard
__('Text', 'resa');
_e('Text', 'resa');

// Mit Escaping (bevorzugt)
esc_html__('Text', 'resa');
esc_attr__('Text', 'resa');

// Plural
_n('Ein Lead', '%d Leads', $count, 'resa');

// Mit Kontext
_x('Lead', 'Tabellen-Header', 'resa');
```

## JavaScript Frontend

```tsx
import { __, _n, _x } from '@wordpress/i18n';

// Standard
__('Text', 'resa');

// Plural
_n('Ein Lead', '%d Leads', count, 'resa');
```

WordPress lädt die JSON-Übersetzungsdateien automatisch für `@wordpress/i18n`.

## Build-Pipeline

```bash
npm run i18n:build
```

Führt vier Schritte aus:

1. **`i18n:pot`** — Extrahiert Strings aus PHP und JS → `languages/resa.pot`
2. **`i18n:update`** — Aktualisiert `.po`-Dateien mit neuen Strings
3. **`i18n:mo`** — Kompiliert `.po` → `.mo` (PHP-Backend)
4. **`i18n:json`** — Generiert `.json` aus `.po` (JavaScript-Frontend)

## Übersetzungsdateien

```
languages/
  resa.pot                          # Template (alle Strings)
  resa-de_DE.po / .mo               # Deutsch (Deutschland)
  resa-de_AT.po / .mo               # Deutsch (Österreich)
  resa-de_CH.po / .mo               # Deutsch (Schweiz)
  resa-de_DE-{hash}.json            # JS-Übersetzungen
```

## DACH-Formate

### Zahlen

| Format    | Beispiel   |
| --------- | ---------- |
| Währung   | 1.250,00 € |
| Dezimal   | 3,5        |
| Tausender | 1.000.000  |
| Fläche    | 85 m²      |
| Prozent   | 12,5 %     |

### Konventionen

- Komma als Dezimaltrennzeichen
- Punkt als Tausendertrennzeichen
- Euro-Zeichen nach dem Betrag mit Leerzeichen
- m² für Quadratmeter
- Formelle Anrede (Sie)
